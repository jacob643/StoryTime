# Story Time — Technical Design Documents

## Overview

Story Time is an adventure typing game driven by an LLM. The player types paragraphs generated by the AI, and their typing speed determines the story's direction — fast typing leads to triumphs, slow typing leads to setbacks.

These documents describe the proposed architecture, design decisions, and implementation plan for rewriting the prototype as a robust, distributable application.

## Key Decisions Summary

| Decision | Choice | Rationale |
|---|---|---|
| Backend language | Python (FastAPI) | Best LLM ecosystem, async-native, easy packaging |
| LLM primary provider | Ollama | Most accessible local LLM runner, auto-discovery via API |
| LLM secondary provider | OpenAI-compatible | Covers custom/remote backends, LM Studio, etc. |
| Frontend | Vanilla HTML/CSS/JS | Zero build step, served by backend, easy to iterate |
| Distribution | PyInstaller → GitHub Releases | Simple .exe for friends, no runtime install needed |
| Story driver | Speed → 5 outcome tiers | Maps directly to existing prompt templates in prototype |

## Current Prototype Issues Addressed

| Issue | Solution |
|---|---|
| Browser CORS blocks Ollama fetch | Backend acts as proxy — browser talks to same-origin server |
| Wrong Ollama API URL | Use `/api/generate` endpoint via httpx on backend |
| LLM output not fed into typing game | Backend returns `next_paragraph` in API response |
| Batch file has syntax errors | Replace with Python entry point + PyInstaller binary |
| No model selection | Provider registry with Ollama auto-discovery |
| All configuration hardcoded | Settings persistence via JSON config file, UI controls |
