Getting Started

Story Time uses an AI model running locally on your computer via Ollama. Follow the steps below to get it running.

1. Install Ollama

  • Windows: Download the installer from ollama.ai/download and run it.
  • macOS: Download the installer from ollama.ai/download or run brew install ollama.
  • Linux: Run curl -fsSL https://ollama.ai/install.sh | sh in a terminal.

2. Start Ollama

After installation, start Ollama from the application menu (Windows/macOS) or run ollama serve in a terminal.

On Windows, Ollama runs as a background system tray app. If the tray icon is missing, launch it from the Start Menu.

3. Download a Model

Open a terminal and run:

ollama pull llama3.2

This downloads a ~2 GB model that works well for story generation. You can use any model from the Ollama library.

4. Make Sure It Runs on Your GPU

Run the model once to load it into memory:

ollama run llama3.2

Then check the GPU usage in another terminal:

ollama ps

Look at the PROCESSOR column. It should show 100% GPU. If it shows a mix like 22%/78% CPU/GPU, the model is splitting across CPU and RAM, which will be very slow.

Fix: Reduce Context Size

If the model isn't fully on GPU, the context window is likely too large for your VRAM. Inside the ollama run session, run:

>>> /set parameter num_ctx 8192
>>> /save llama3.2

Then quit (/bye), stop the model, and re-run:

ollama stop llama3.2
ollama run llama3.2

Verify with ollama ps that it now shows 100% GPU.

Check Your VRAM

NVIDIA users: Run nvidia-smi to see how much VRAM is used vs available.
AMD / Intel users: Use rocm-smi (AMD) or check Task Manager (Windows) / intel_gpu_top (Linux).

Each model's size plus its context window (KV cache) must fit in your GPU's VRAM. As a rough guide:

  • 8 GB VRAM: 3B–8B models with 4K–8K context
  • 12 GB VRAM: 7B–13B models with 8K–16K context
  • 24+ GB VRAM: Most models fit comfortably

5. Open Story Time

Start the game server:

python -m backend.main

Or use the convenience script:

backend/run.bat

Your browser should open automatically. If not, navigate to http://127.0.0.1:8000.

Type a story premise and press Send to begin!

Troubleshooting

  • "Connection refused" / 503 error: Ollama isn't running. Start it as described in step 2 and wait for the tray icon to appear.
  • Story generation is very slow: Check ollama ps — if you see less than 100% GPU, reduce num_ctx as described above.
  • Ollama runs out of memory: Try a smaller model like llama3.2:1b or qwen2.5:0.5b.
